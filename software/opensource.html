Evolution: Workaround Improper GB-2312 Mail Encodings
https://bugzilla.gnome.org/show_bug.cgi?id=666896
Submitted: December 27, 2011
Resolved: April 11, 2012

Lulu discovered that Evolution didn't set the correct character encoding all the time for some Chinese encodings. The problem actually stems from the fact that there are 2 Chinese character encodings, one which is a superset of the other. Some clients would specify the subset as the encoding while including characters only valid in the superset. Evolution would correctly print out garbage in this case, but as many other mail clients seem to have a workaround for this specific case it seemed reasonable for Evolution to have the workaround as well.

I submitted a patch that changed the encoding to the superset whenever the subset was detected. This was fairly simple because Evolution already had an encoding mapping list, so I just changed it.
